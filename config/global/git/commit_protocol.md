# Commit Protocol

Commit messages should explain why the decision was made and include useful trailers when they add context.

Recommended format:

```text
<intent line: why the change was made>

Constraint: <external constraint>
Rejected: <alternative> | <reason>
Confidence: <low|medium|high>
Scope-risk: <narrow|moderate|broad>
Tested: <verification evidence>
Not-tested: <known gap>
```
